# Controlled Pilot RACI

| Activity | Management/Admin | IT | Quality | Pilot coordinator | Pilot users |
|---|---|---|---|---|---|
| Approve pilot scope | A | C | C | R | I |
| Approve user limit | A | C | C | R | I |
| Review technical evidence | I | A/R | C | C | I |
| Review OVR confidentiality | I | C | A/R | C | I |
| Configure pilot users | C | A/R | C | R | I |
| Run UAT scenarios | I | C | C | R | R |
| Record issues | I | C | C | A/R | R |
| Respond to access incident | I | A/R | C | C | I |
| Respond to confidentiality incident | A | R | A/R | C | I |
| Approve pilot continuation after critical issue | A | C | C | R | I |
| Close pilot | A | C | C | R | I |

Legend:

```text
R = Responsible
A = Accountable
C = Consulted
I = Informed
```

## Approval accountability

The approval files require real accountable humans. The system must not infer approval from passing scripts.
