# Pilot Evidence Capture Operations

## Objective

Standardize how pilot evidence is captured during daily controlled-pilot execution.

## Evidence channels

- Operator daily checklist
- Pilot issue log
- Access review checklist
- UAT scenario results
- Staging smoke test evidence
- Restore dry-run evidence
- Security and confidentiality confirmation status

## Minimum daily evidence

1. Date and operator name.
2. Active pilot users count.
3. Modules tested.
4. Issues opened and closed.
5. Any access or confidentiality exception.
6. Go/no-go notes.

## Prohibited evidence

- Real patient identifiers.
- Confidential OVR details.
- Screenshots containing sensitive personal data.
- Production system secrets.
