# Approval Entry QA Guide

Use this guide before running the final proof.

## Pilot signoff file

File: `release/v674/approvals/pilot-signoff.json`

Check that Management/Admin, IT, and Quality each have a real name, real role, date, decision, and controlled-pilot scope.

## Confidentiality file

File: `release/v674/approvals/ovr-confidentiality-confirmation.json`

Check that IT and Quality confidentiality reviewers have real reviewer details, date, decision, controlled-pilot scope, and all required confidentiality statements set to true.

## Do not

- Do not use placeholder approvers.
- Do not sign on behalf of reviewers.
- Do not expand beyond 15 pilot users.
- Do not permit real patient identifiers.
- Do not include confidential OVR details.
