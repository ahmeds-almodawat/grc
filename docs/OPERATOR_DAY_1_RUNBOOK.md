# Operator Day 1 Runbook

Use this on the first day of pilot operation.

## Morning checks

1. Open Home.
2. Open Production Finish Center.
3. Check blocker count.
4. Open Operations & Notifications Center.
5. Review overdue actions and pending approvals.
6. Open OVR page and confirm no major unreviewed OVR is stuck.
7. Run an export package from the Export Center.

## First user verification

Test these accounts:

- Executive account sees command center and company-wide dashboards.
- Department manager sees only department scope.
- Normal employee sees only assigned work and own submitted OVRs.
- Quality user can review/close OVRs.
- Auditor can manage audit findings but not unrelated admin controls.

## First workflow test

1. Create a project.
2. Add a milestone.
3. Add a task with owner and due date.
4. Upload evidence.
5. Request approval.
6. Approve using a different user.
7. Close after accepted evidence.

## First OVR test

1. Submit an OVR as employee.
2. Supervisor/HOD adds investigation.
3. Quality confirms severity.
4. Create corrective action project.
5. Upload evidence.
6. Quality closes.
7. Confirm it appears in OVR risk indicators.

## End of day checks

1. Export daily backup package.
2. Record issues in the defect log.
3. Record unresolved blockers in the Production Finish Center.
4. Send executive summary to sponsor.
