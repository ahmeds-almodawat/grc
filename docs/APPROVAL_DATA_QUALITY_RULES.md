# Approval Data Quality Rules

## Rules

- Do not use placeholders.
- Names must identify real approvers.
- Roles must be real organizational roles.
- Dates must be `YYYY-MM-DD` and not in the future.
- Decisions must use allowed values.
- Scopes must clearly state controlled internal pilot boundaries.
- Evidence review flags must only be true after real review.
- Confidentiality statements must only be true after real review.

## Forbidden shortcuts

- Fake approvers.
- Auto-filling approvals.
- Copying placeholder names.
- Expanding the scope to production.
- Using real patient data or confidential OVR details.
