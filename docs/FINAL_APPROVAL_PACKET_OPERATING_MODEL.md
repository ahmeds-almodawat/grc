# Final Approval Packet Operating Model

## Operating principle

Approvers should review evidence before completing signoff. The platform should remain in a controlled-pilot-only state until approvals are real, complete, and validated by the strict proof chain.

## Packet sections

1. Technical proof summary
2. Restore proof summary
3. Runtime bridge/persona proof summary
4. OVR confidentiality constraints
5. Controlled pilot scope
6. Approver responsibility matrix
7. Final command sequence after signoff

## Prohibited shortcuts

- Do not use placeholder names.
- Do not backdate approvals.
- Do not exceed the controlled pilot user limit.
- Do not include real patient identifiers.
- Do not include confidential OVR details.
- Do not mark production ready.
