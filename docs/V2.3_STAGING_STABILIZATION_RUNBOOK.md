# v2.3 Ultra Staging Stabilization Runbook

This patch turns the latest honest assessment into operational controls. It does **not** add another business module. It adds the controls needed before real staff rollout.

## Main purpose

Use this patch to prove that the platform is ready for staging and later production:

1. Run all migrations from `001` to `025` on a clean Supabase staging project.
2. Seed the v2.3 staging validation defaults.
3. Test real RLS/persona access with actual test users.
4. Audit Arabic/English coverage and RTL readiness.
5. Prepare load testing for 1,000 users, 50 departments, and high-volume work records.
6. Define a real production backup strategy beyond browser export.
7. Track defects that must be fixed, accepted, or documented before production.

## New pages

- Staging Validation Center
- RLS Persona Lab
- Translation Coverage Center
- Load Seed Center

- Migration Runbook Center

## Suggested validation order

1. Apply patch files to the existing project.
2. Run `npm run typecheck` and `npm run build`.
3. Apply migration `025_staging_validation_consolidation.sql` after migration `024`.
4. Open **Staging Validation Center**.
5. Press **Seed defaults** if records are not visible.
6. Review blockers and warnings.
7. Open **RLS Persona Lab** and create actual test users for each persona.
8. Open **Translation Coverage Center** and complete Arabic/RTL review.
9. Open **Load Seed Center** and execute only in staging, not production.
10. Open **Production Backup Strategy** and document database, storage, and auth recovery.
11. Open **Migration Runbook Center** and mark each migration as verified after testing.

## Production rule

Do not deploy to all employees until the following are green or formally accepted:

- Zero critical RLS failures.
- All required migrations verified.
- Restore dry-run completed.
- Arabic/RTL review for active workflows completed.
- Load test accepted for 1,000-user target.
- Critical defect log cleared or accepted by management.

## Notes

The production backup strategy added here is governance metadata. It does not automatically create server-side Supabase backups. Production backup still requires Supabase/DB/storage-level configuration by IT or your hosting administrator.
